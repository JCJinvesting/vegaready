# Credit-markets feed sourcing plan: automation, licensing, and failover

## Executive decision

For a public market-intelligence page, the clean engineering answer is not the same as the clean data-access answer: FRED is technically excellent for the two ICE/BofA OAS series, but the ICE terms on the FRED series pages say the end-of-day index values, returns, and statistics are provided for internal use only and may not be published, distributed, or furnished to third parties without ICE Data’s prior written approval ([FRED BAMLC0A0CM](https://fred.stlouisfed.org/series/BAMLC0A0CM), [FRED BAMLH0A0HYM2](https://fred.stlouisfed.org/series/BAMLH0A0HYM2)). FRED’s general terms also say that before using third-party-owned series for anything other than personal use, the user must contact the data owner for permission, and FRED cannot grant that permission ([FRED legal terms](https://fred.stlouisfed.org/legal/)). Therefore, use FRED for internal monitoring/prototyping of ICE series, but do not display ICE/BofA values publicly unless you get a license from ICE or replace the tile with a licensed widget/paid feed ([FRED legal terms](https://fred.stlouisfed.org/legal/)).

The most automatable free/public tiles are the SEC quarterly corporate-bond-offerings aggregate and, with caveats, World Bank GEM / DataBank emerging-market financial data if the user’s existing EMBI source is from World Bank and metadata permits reuse ([SEC corporate bond offerings](https://www.sec.gov/data-research/statistics-data-visualizations/corporate-bond-offerings), [World Bank dataset terms](https://www.worldbank.org/en/about/legal/terms-of-use-for-datasets)). The least responsible “free” tiles are HY distress ratio, CDX IG/HY live or delayed levels, Morningstar/LSTA loan index level/spread, and any ICE/BofA/FRED top-level index value for public display without vendor permission, because the available public pages are either non-programmatic, expressly restricted, or publish only narrative/monthly snapshots with proprietary underlying data ([Eaton Vance high-yield monitor](https://www.eatonvance.com/content/dam/im/assets/publication/thought-leadership/article/article_highyieldmarketmonitor_q12026.pdf?1778544000225), [S&P DJI terms](https://www.spglobal.com/spdji/en/terms-of-use/), [Morningstar leveraged-loan methodology](https://indexes.morningstar.com/api/docs/69bd3e9250ca695336d4045f)). My recommendation is to classify each tile as `licensed-live`, `public-lagged`, `manual-public-snapshot`, or `feed pending`, and to render the value only when the stored source license flag is `display_allowed=true`.

## FRED implementation detail

FRED’s observations endpoint is `https://api.stlouisfed.org/fred/series/observations`, requires `api_key` and `series_id`, defaults to XML, and returns JSON when `file_type=json` is supplied ([FRED series observations API](https://fred.stlouisfed.org/docs/api/fred/series_observations.html)). FRED observation responses include top-level fields such as `realtime_start`, `realtime_end`, `observation_start`, `observation_end`, `units`, `output_type`, `file_type`, `order_by`, `sort_order`, `count`, `offset`, and `limit`, plus an `observations` array whose objects contain `realtime_start`, `realtime_end`, `date`, and string-valued `value` fields ([FRED series observations API](https://fred.stlouisfed.org/docs/api/fred/series_observations.html)).

```bash
curl -sS 'https://api.stlouisfed.org/fred/series/observations' \
  --get \
  --data-urlencode "series_id=BAMLH0A0HYM2" \
  --data-urlencode "api_key=$FRED_API_KEY" \
  --data-urlencode "file_type=json" \
  --data-urlencode "observation_start=2024-01-01" \
  --data-urlencode "sort_order=desc" \
  --data-urlencode "limit=10"
```

Use the metadata endpoint `https://api.stlouisfed.org/fred/series` to validate `units`, `frequency`, `last_updated`, and `notes` before accepting a new series into production ([FRED series metadata API](https://fred.stlouisfed.org/docs/api/fred/series.html)). The metadata endpoint requires `api_key` and `series_id`, and its JSON response contains a `seriess` array with fields such as `id`, `title`, `observation_start`, `observation_end`, `frequency`, `units`, `seasonal_adjustment`, `last_updated`, and `notes` ([FRED series metadata API](https://fred.stlouisfed.org/docs/api/fred/series.html)).

```bash
curl -sS 'https://api.stlouisfed.org/fred/series' \
  --get \
  --data-urlencode "series_id=BAMLC0A0CM" \
  --data-urlencode "api_key=$FRED_API_KEY" \
  --data-urlencode "file_type=json"
```

FRED API keys are 32-character lowercase alphanumeric strings, all web-service requests require an API key, and developers are told to request a distinct API key for each application ([FRED API key docs](https://fred.stlouisfed.org/docs/api/api_key.html)). FRED’s official API terms do not publish a fixed numeric rate limit, but they reserve the right to impose or adjust bandwidth and transaction limits, and the error docs list HTTP 429 `Too Many Requests` for rate limiting ([FRED API terms](https://fred.stlouisfed.org/docs/api/terms_of_use.html), [FRED API errors](https://fred.stlouisfed.org/docs/api/fred/errors.html)).

Recommended FRED refresh cadence: poll the two OAS series once per business day at 10:15 Central time, then retry at 12:15 and 16:15 only if the latest observation date is older than the expected prior business day; this produces at most six observation calls per day for two series, far below any plausible practical API limit and small enough to fit a simple daily cache ([FRED BAMLC0A0CM](https://fred.stlouisfed.org/series/BAMLC0A0CM), [FRED BAMLH0A0HYM2](https://fred.stlouisfed.org/series/BAMLH0A0HYM2), [FRED API errors](https://fred.stlouisfed.org/docs/api/fred/errors.html)). Treat `.` values as missing, parse numeric strings into decimals, store the `realtime_start`/`realtime_end` vintage fields with every observation, and alert if `last_updated` changes but the latest valid observation date does not advance for two expected business days ([FRED series observations API](https://fred.stlouisfed.org/docs/api/fred/series_observations.html), [FRED series metadata API](https://fred.stlouisfed.org/docs/api/fred/series.html)).

## Reliability pattern

Use a two-layer model: `raw_observation` stores every vendor response with source URL, retrieval timestamp, HTTP status, ETag/Last-Modified if present, and license class; `display_metric` is materialized only if the observation passes freshness, unit, license, and definition checks. For FRED, a value should pass only if `series_id` matches the allow-list, `units == Percent`, `frequency == Daily`, latest date is within the expected business-day lag, and legal status is either `internal_only` or `display_allowed`; for the public site, `internal_only` must suppress rendering even if the value is fresh ([FRED series metadata API](https://fred.stlouisfed.org/docs/api/fred/series.html), [FRED BAMLC0A0CM](https://fred.stlouisfed.org/series/BAMLC0A0CM)).

For non-API public snapshots such as Fitch/ABI, Eaton Vance/J.P. Morgan, Nuveen, Invesco, and SIFMA, do not treat page availability as display permission, because multiple documents explicitly say underlying index data is used with permission or may not be copied/distributed without vendor approval ([Eaton Vance high-yield monitor](https://www.eatonvance.com/content/dam/im/assets/publication/thought-leadership/article/article_highyieldmarketmonitor_q12026.pdf?1778544000225), [Invesco loan market snapshot](https://www.invesco.com/content/dam/invesco/be/en/pdf/senior-secured-loans/Invesco-us-loan-markets-snapshot.pdf), [SIFMA terms](https://www.sifma.org/terms-of-use/)). The safest fallback UX is a dated source link with no numeric value, such as “Latest public discussion: Fitch monthly monitor via ABI, Sep. 18, 2025,” or a tile state like “licensed feed required,” because that is more honest than publishing an unlicensed or stale number ([ABI summary of Fitch report](https://www.abi.org/node/1000963), [Fitch Group terms](https://www.fitch.group/terms-of-use)).

## Explicit no-go list for free automated public tiles

Keep these as `feed pending` unless you obtain a redistribution license: ICE/BofA IG OAS and HY OAS from FRED; HY distress ratio; TTM HY default rate from Fitch/Moody’s/S&P research; CDX IG/HY levels; Morningstar/LSTA loan index level/spread; and SIFMA IG/HY issuance if your use is commercial/public redistribution rather than a summarized link with permission ([FRED BAMLC0A0CM](https://fred.stlouisfed.org/series/BAMLC0A0CM), [FRED BAMLH0A0HYM2](https://fred.stlouisfed.org/series/BAMLH0A0HYM2), [Fitch Group terms](https://www.fitch.group/terms-of-use), [Moody’s terms](https://www.moodys.com/web/en/us/legal/terms-of-use.html), [S&P DJI terms](https://www.spglobal.com/spdji/en/terms-of-use/), [Morningstar leveraged-loan methodology](https://indexes.morningstar.com/api/docs/69bd3e9250ca695336d4045f), [SIFMA terms](https://www.sifma.org/terms-of-use/)).

If you want a small set of public, low-risk placeholders, use SEC quarterly corporate-bond-offerings totals as a legally cleaner aggregate issuance tile, use FRED only for public-domain or citation-required non-proprietary macro series after checking each series’ copyright line, and use your already-licensed EMBI source if your contract permits public display with attribution ([SEC corporate bond offerings](https://www.sec.gov/data-research/statistics-data-visualizations/corporate-bond-offerings), [FRED legal terms](https://fred.stlouisfed.org/legal/), [World Bank dataset terms](https://www.worldbank.org/en/about/legal/terms-of-use-for-datasets)).
