## 4. Comprehensive Analysis

**High-Confidence Findings.** All three models converge on a clear conclusion: of the four tiles, **CDX.NA.IG/HY is the easy one** thanks to CFTC Part 43 real-time public dissemination through DTCC's PPD platform. The other three tiles have no comparable regulatory backstop and require either ETF-derived proxies or polled fund-manager publications. Claude Opus 4.8 and GPT-5.5 both reverse-engineered the DTCC API surface; Gemini 3.1 Pro identifies the same channel at a higher level. All three reject Investing.com / TradingView "CDX" tickers as misleading site-specific constructs, and all three reject FRED for any of the four canonical figures (the aggregate HY OAS is the closest FRED gets, but it is not the distress ratio). For the EMBI Global Diversified tile, all three concur there is no clean free feed; the EMB ETF is the obvious proxy but must be labeled honestly because EMB tracks **EMBI Global Core**, not Diversified — a divergence that is more than cosmetic given the issuer-capping differences between Core and Diversified.

**Areas of Divergence.** The most consequential disagreement is on CDX.NA.HY's pricing convention in the SDR feed. The original brief assumed HY quotes in price (a market convention rooted in the 500bp coupon). Gemini repeats that prior. Claude Opus 4.8, who actually downloaded and parsed the 12-Jun-2026 CREDITS file, found the `Price` column **empty across all 247 HY rows** and `Spread-Leg 1` cleanly populated — concluding that the U.S. SDR disseminates HY as spread. GPT-5.5, working from the dashboard's JavaScript source code, describes both price and spread fields as available in the schema and recommends preferring price when present. The most defensible reading is Claude's: on the verified day, HY was disseminated as spread. The production pipeline should handle both cases but compute from spread as the primary, with spread→price conversion (via the standard CDX upfront formula given coupon and index factor) as a derived display. The second material disagreement is the Morningstar LSTA broad index level: Claude claims the public screener at `indexes.morningstar.com/indexes/screener?filterPattern=Leveraged+Loan` returns 4,351.35 daily with no login; GPT-5.5 visited the broad-index detail page (`FS0000HS4A`) and found it explicitly states performance data is unavailable. These are not strictly contradictory — Claude used the screener listing view, GPT-5.5 used the detail view — but a brief manual verification of the screener page before relying on it is warranted.

The third axis of disagreement is philosophical: for HY distress and EMBIGD, **does the PRIMARY tile carry the named, lagged figure or the daily-fresh proxy?** Claude argues fidelity first — show the actual J.P. Morgan / FridsonVision distress ratio from the latest publication, and only fall back to HYG-derived approximations if the publication is stale. Gemini and GPT-5.5 reverse the order — show the live ETF-derived proxy daily, with the lagged publication as confirmation. Both stances are defensible under the stated display policy ("publish any figure I can reliably fetch, with full source attribution"); the deciding factor is the desk's read on what users expect when they look at a tile labeled "HY distress ratio." If the label is canonical, primary should be the named publication. If the label can read "HY distress proxy" honestly, primary can be daily and ETF-derived. A reasonable compromise is to display both, side-by-side, with the labels disambiguated.

**Unique Insights Worth Noting.** Claude's empirical validation of the entire DTCC pipeline — extracting actual IG/HY median spreads from the file — is the highest-value single contribution of this council; it converts the CDX tile from a research bet into an engineering task with known ground-truth values. The second-most-valuable unique finding is GPT-5.5's identification of the **BlackRock `varnish-api` JSON product-data API**, which is almost certainly the right production replacement for the deprecating legacy iShares ajax CSV endpoint that Claude observed to be consent-wrapped. Together, these two findings define the modern operational path: DTCC for CDX, BlackRock varnish-api for HYG and EMB holdings/fundamentals. GPT-5.5 also surfaces that the Morgan Stanley/Eaton Vance EVLN monthly review PDF contains the broad LSTA spread-to-3-year explicitly — the cleanest single fallback for that tile's spread if the Morningstar screener path turns out to be Loan-100-only as GPT-5.5 suggests.

**Recommendations.** Build the CDX tile first using Claude's verified DTCC extraction logic (UPI filter + dominant-expiration cluster + median `Spread-Leg 1`), with GPT-5.5's `/ppd/api/ticker/CFTC/CREDITS` and slice endpoints as the operational polling targets at ~120-second cadence. Honest label: "CDX.NA.IG / CDX.NA.HY 5Y on-the-run — median SDR trade print (spread, bp), cleared, 15-min delayed; DTCC public dissemination, not Markit/S&P fixing." For the HY distress and EMBIGD tiles, adopt a two-row display: top row is the named figure from the latest scheduled publication with its as-of date (FridsonVision/Eaton Vance for HY; TCW/Fort Washington/SSGA for EMBIGD), bottom row is a daily ETF-derived proxy from the BlackRock varnish-api with the formula spelled out — both rows visible, no merging. For the Morningstar LSTA tile, **verify the screener claim manually first**; if it holds, use it daily for the level and use the EVLN monthly PDF for the spread-to-3-year. If the screener turns out to be Loan-100-only, fall back fully to the Morgan Stanley monthly review with month-end labeling. JNK is not a viable distress proxy — drop it from the design. The full per-metric URLs, fields, fragility notes, and labels are in the three individual reports; the synthesis above identifies which to trust on each contested point.
