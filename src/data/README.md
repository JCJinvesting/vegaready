# VegaReady analysis data — single source of truth

**Rule: all analysis-page content lives in `src/data/`. Pages import it. Never embed data arrays inside `.astro` frontmatter.**

A previous restructure silently reverted four content fixes because the data was hand-embedded in the page templates and duplicated across two page trees (`/scenarios/*` and the markets-first pages). When the new pages were branched from an older commit, the corrections were lost with no error. Single-sourcing the data here prevents that class of regression: there is exactly one place to edit each dataset, and a restructure carries it forward by import.

## Files

| File | Feeds | Exports |
|------|-------|---------|
| `analysis/transmission.js` | `/transmission` (Sector Cascade) | `scenarios` (5-layer), `sectors` (12) |
| `analysis/exposure.js` | `/exposure` (Regional Impact) | `countries` (12), `winnerLosers` (12) |
| `analysis/outlook.js` | `/outlook` (Gap Dynamics) | `flows` (6), `crossFlow` |
| `analysis/connections.js` | `/connections` | `intro`, `themes` (6 cross-cutting threads) |
| `analysis/freshness.js` | freshness strip in `AnalysisLayout` (all areas) | `freshness` (scenario status + live anchors, as-of date) |
| `iwt-bundle.json` | dashboard (`window.__IWT_DATA__`) | generated by `scripts/build-data.cjs` from IranWarTracker JSON |

Pages import like: `import { sectors } from '../../data/analysis/transmission.js';`

## How to update content

1. Edit the relevant `src/data/analysis/*.js` module. **Do not edit the data inside the `.astro` page** — the page only renders it.
2. Update `freshness.js` (status chips + anchors + `asOf`) whenever the live situation moves — this is the site-wide "as-of" strip.
3. Commit + push; Cloudflare rebuilds.

## Provenance — where the research actually lives

The deep, fully-sourced research is in the **IranWarTracker** repo: `data/cascades/{sectors,regions,dynamics,scenario-status}.json` (carrying `sources`, `confidence`, `dataQualityExceptions`, `verificationLog`). The `src/data/analysis/*.js` modules here are the **condensed display layer** derived from those canonical files. When you correct a fact, fix it in the IWT canonical JSON **and** the matching display module.

## Recommended next step (full automation)

To make IWT the *single* source end-to-end, extend `scripts/build-data.cjs` to regenerate `src/data/analysis/*.js` from `IranWarTracker/data/cascades/*.json` at build time (mapping the deep schema → the flat card fields). Then the canonical JSON is the only thing anyone edits and the site regenerates on every build. Verify with a local `npm run build` before relying on it.
